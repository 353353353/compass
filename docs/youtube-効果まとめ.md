# YouTube 効果まとめ（ルールブック 兼 自動化の設計図）

> このドキュメントの目的は2つ。
> 1. **編集中に「ここ何すればいい?」で手が止まらないようにする**（craft＝作り方の知識）
> 2. **将来 Claude Code で「場面→効果」を自動提案・自動適用する時の設計図にする**（automation の仕様）
>
> だから書き方のルール：**「なんとなくカッコいい」ではなく「どの場面で・何を・なぜ」当てるかを言葉にする**。
> 言葉になったルールだけが、後で自動化できる。

## 1. 視聴維持の原則（このチャンネル用）

飽きさせない＝**「変化」と「緊張と緩和」を切らさない**こと。掛け合い動画は特にテンポが命。

1. **冒頭5〜15秒で"一番の驚き"を先に出す**
   - 例：「コード1行も書いてないんだけど、これ作った」（＝SHADOW OPS scene01）
   - 説明から入らない。結論・オチ・一番ヤバい絵を先に見せて「どうやって?」を作る。
2. **3〜5秒に1回、画面に変化を作る**
   - 立ち絵の動き／テロップ／ズーム／表情差分。無変化の"死んだ時間"を消す。
3. **ボケ→ツッコミを1つのリズム単位にする**
   - ボケの直後に必ず「間」か「SE」か「ツッコミテロップ」。オチを置き去りにしない。
4. **問題→解決の緊張と緩和を1本に最低1回**
   - エラー発生（緊張）→ 修正で完璧に直る（緩和＝カタルシス）。SHADOW OPS scene04 がこれ。
5. **場面転換ごとにBGM/SEを変えて「区切り」を体感させる**
   - 章が変わったと視聴者に分からせると、離脱の踏みとどまりになる。
6. **間延び（無音・言い淀み・待ち時間）は削る**
   - AI実演の"生成待ち"は早送り or カット or テロップで埋める。
7. **クライマックスに向けて演出を盛る**
   - 終盤（ボス対決など）で効果音・エフェクトの密度を上げる。ラストにCTA。

## 3. キャラ別・演出ルール

掛け合いは「どっちが何担当か」を固定すると効果が自動で決まる。

### ゆうにゃん（満別花丸）＝ボケ・天才・暴走役
- ぶっ飛んだ提案の時：立ち絵アップ＋キラキラ／ドヤ顔差分
- 暴走（勝手にスマホ使う等）：やらかしSE＋軽いフラッシュ

### けろみん（春日部つむぎ）＝ツッコミ・AI信仰役
- ツッコミ：素早いカット＋ビシッSE＋大テロップ
- AI（クロちゃん）礼賛：後光エフェクト／キラーンSE（ネタとして大げさに）

## 4. 実例：SHADOW OPS を効果マッピングしてみる

> お手本動画を"効果の言葉"に翻訳したもの。これがそのまま次回作のテンプレになる。

| シーン | 内容 | 当てる効果（この表の適用） |
|---|---|---|
| scene01 | 冒頭フック。コード0行を明かす | 冒頭フック行：完成ゲーム画面を先出し＋「コード0行」大テロップ＋掴みBGM |
| scene02 | 二段階の驚き（勝手にスマホ） | 驚き・二段構え行：ズーム→ズーム＋ドン→ドドン＋「!?」／やらかしSE |
| scene03 | 「射撃ゲーム作って」としゃべっただけ | 説明行：音声入力の実演キャプチャ＋要点テロップ＋ピコン |
| scene04 | エラー（ボス全員同じ顔）→修正で完璧 | 緊張→緩和：エラー音＋赤み → 修正で明転＋成功音（本編の山場①） |
| scene05 | 実演。「影分身アタック」誕生 | 実演行：プレイ画面＋要所ズーム＋実況テロップ／必殺技名を大きく |
| scene06 | ボス対決クライマックス。「派手なウニ」 | クライマックス行：エフェクト密度UP＋派手SE連打＋盛上BGM（山場②） |
| scene07 | まとめ＋登録CTA | CTA行：締めBGM＋登録ボタン演出＋矢印 |

**気づき：山場が2つ（scene04 修正カタルシス / scene06 クライマックス）ある。**
これが視聴維持の背骨。次回作も「中盤にカタルシス1つ・終盤に盛り上がり1つ」を最低ラインにする。

## 6. これがどう「自動化」につながるか（次のステップ）

このルールブックが育つと、次の自動化が現実的になる：

1. **台本→効果指示の自動生成**
   - 台本の各行に「これは どの場面タイプ か」をタグ付け → 第2章の表を引いて「当てる効果」を自動で並べる。
   - Claude Code に台本を渡す → 「scene04はエラー→修正だから、エラー音＋赤み→成功音、と指示書を出す」まで自動化。
2. **素材の割り当て自動化**
   - 第5章の素材リストが揃えば、「驚き＝この効果音ファイル」と対応させ、指示書に実ファイル名まで入れられる。
3. **YMM4 への橋渡し（将来）**
   - YMM4のプロジェクト構造を調べて、指示書→実配置まで自動化できるか検証する（要調査。ここは"やれるか未確定"）。
